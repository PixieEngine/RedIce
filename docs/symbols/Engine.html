Class Detail
Engine(I)
The Engine controls the game world and manages game state. Once you set it up and let it run it pretty much takes care of itself. You can use the engine to add or remove objects from the game world. There are several modules that can include to add additional capabilities to the engine. The engine fires events that you may bind listeners to. Event listeners may be bound with engine.bind(eventName, callback)

static Engine.HUD
The HUD module provides an extra canvas to draw to. GameObjects that respond to the drawHUD method will draw to the HUD canvas. The HUD canvas is not cleared each frame, it is the responsibility of the objects drawing on it to manage that themselves.
static Engine.SaveState
The SaveState module provides methods to save and restore the current engine state.
static Engine.Selector
The Selector module provides methods to query the engine to find game objects.
static Engine.Shadows
The Shadows module provides a lighting extension to the Engine. Objects that have an illuminate method will add light to the scene. Objects that have an true opaque attribute will cast shadows.

The Tilemap module provides a way to load tilemaps in the engine.

{Array} find(selector)
Get a selection of GameObjects that match the specified selector criteria. The selector language can select objects by id, class, or attributes. To select an object by id use "#anId" To select objects by class use "MyClass" To select objects by properties use ".someProperty" or ".someProperty=someValue" You may mix and match selectors. "Wall.x=0" to select all objects of class Wall with an x property of 0.
