Engine(I)
The Engine controls the game world and manages game state. Once you set it up and let it run it pretty much takes care of itself. You can use the engine to add or remove objects from the game world. There are several modules that can include to add additional capabilities to the engine. The engine fires events that you may bind listeners to. Event listeners may be bound with engine.bind(eventName, callback)
Parameters:
I Object
Instance variables of the engine

This module clears or fills the canvas before drawing the scene.

The Developer module provides a debug overlay and methods for debugging and live coding.

The FPSCounter module tracks and displays the framerate. 
window.engine = Engine
  ...
  includedModules: ["FPSCounter"]
  FPSColor: "#080"
Engine.HUD static
The HUD module provides an extra canvas to draw to. GameObjects that respond to the drawHUD method will draw to the HUD canvas. The HUD canvas is not cleared each frame, it is the responsibility of the objects drawing on it to manage that themselves.
Engine.Joysticks static
The Joysticks module gives the engine access to joysticks. 
  # First you need to add the joysticks module to the engine
  window.engine = Engine
    ...
    includedModules: ["Joysticks"]
  # Then you need to get a controller reference
  # id = 0 for player 1, etc.
  controller = engine.controller(id)

  # Point indicating direction primary axis is held
  direction = controller.position()

  # Check if buttons are held
  controller.actionDown("A")
  controller.actionDown("B")
  controller.actionDown("X")
  controller.actionDown("Y")

The Shadows module provides a lighting extension to the Engine. Objects that have an illuminate method will add light to the scene. Objects that have an true opaque attribute will cast shadows.

find(selector)
Get a selection of GameObjects that match the specified selector criteria. The selector language can select objects by id, class, or attributes. Note that this method always returns an Array, so if you are trying to find only one object you will need something like engine.find("Enemy").first(). 
    player = engine.add
      class: "Player"

    enemy = engine.add
      class: "Enemy"
      speed: 5
      x: 0

    distantEnemy = engine.add
      class "Enemy"
      x: 500

    boss = engine.add
      class: "Enemy"
      id: "Boss"
      x: 0

    # to select an object by id use "#anId"
    engine.find "#Boss"
    # => [boss]

    # to select an object by class use "MyClass"
    engine.find "Enemy"
    # => [enemy, distantEnemy, boss]

    # to select an object by properties use ".someProperty" or ".someProperty=someValue"
    engine.find ".speed=5"
    # => [enemy]

    # You may mix and match selectors.
    engine.find "Enemy.x=0"
    # => [enemy, boss] # doesn't return distantEnemy
Parameters:
selector String
Returns Array :
An array of the objects found

    canvas.drawText

overlay() event
Called after the engine draws. The current camera transform is not applied. This is great for adding overlays. 
  engine.bind "overlay", (canvas) ->
    # print the player's health. This will be
    # positioned absolutely according to the viewport.
    canvas.drawText
      text: "HEALTH:"
      position: Point(20, 20)

    canvas.drawText
      text: player.health()
      position: Point(50, 20)
update() event
Called when the engine updates all the game objects.
